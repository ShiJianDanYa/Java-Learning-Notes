# 2— 本质 意义 解决方法
## 1— 方法
方法：程序中的独立功能，也是最小的执行单元

独立功能：一个方法只负责完成一件具体、单一、不依赖其他无关逻辑的事情 （一个方法只完成一件事）
执行单元：方法是一个打包的整体代码，要么都不执行，要么就从上到下把方法所有代码都执行完毕(要么全部执行,要么全不执行)

优点 提高了程序的复用性和可维护性
## 2— 方法的格式
```java
public static 返回值类型 方法名(参数1,参数2...){
	方法体;
	return 返回值;
}
```
开始推导
方法是一个帮你办事的人
你需要 让他帮你计算 两数之和(方法体)
喊他名字 需要调用它 （需要方法名）  
告诉他是那两个数(参数列表)
最后告诉你结果(返回值)

而他需要做什么
1. 接收数据  （）里面需要参数
2. 计算结果   方法体
3. 返回结果   return

## 4— 定义方法的诀窍
需要回答三个问题
 1. 那些代码写到方法中  观察在大段的代码当中， 反复使用的独立功能是什么？反复使用+独立功能
 2. 方法的形参怎么确定 这个独立功能，需要什么才能完成
 3. 方法的返回值如何确定 方法的调用处，是否需要这个独立功能的结果继续做其他事情 没有则是void

# 3— 注意事项
1. 方法实参形参，返回值和返回值类型要一一对应
2. 方法之间是平行的（包括main方法）所以方法的位置是main方法外，类的里面
3. 方法需要调用才能执行

方法没有返回值
则方法的返回值类型写void


# 1— 知识扩展
return 作用
1. 结束方法的运行
2. 将结果返回给方法的调用处

vodi 省略
没有返回值，可以省略return不写
如果不省略，return后面不能写具体数据，仅表示结束方法
