## 1— 配置环境变量步骤
以前下载的老版本的DK是没有自动配置的，而且自动配置的也只包含了4个工具而已，所以我们需要删掉已经配置完毕的，再次重新配置Path环境变量。
1. 打开搜索栏,输入环境变量
2. 配置JAVA_HOME
3. 配置path
4. 检测是否安装成功

第四步 检测是否安装成功
按下win+R，
在最下角出现的对话框中输入cmd
![[配置环境变量_2026-03-30_18-01-33.png]]
查看JDK版本
再弹出的黑屏窗口中输入命令：java -version
注意1：java后面有一个空格和横杠，然后再输入version注意2：所有的符号都是英文状态下的
注意3：检查输入是否正确，如正确，按下回车
回车之后，如出现以下信息，则代表安装并设置成功
![[配置环境变量_2026-03-30_18-12-01.png]]
解释信息
```java
javaversion"17.0.12"2024-07-16LTS// 本次使用java的大版本为17，小版本为12，表示把17的版本，更新12次的小版本，更新时间为：2024年7月16日，是LTS正式版
Java(TM) SE Runtime Environment
：(bui1d17.0.12+8-LTS-286)//Java标准版运行环境的构建
号为17。0。12+8-LTS-286。286是一个补丁号。
Java HotSpot(TM) 64-Bit Server VM (build 17.0.12+8-LTS-286, mixed mode,sharing）//这表示Java使用的是HotSpot虚拟机，这是一个64位的服务器版虚拟机。构建号与运行时环境相同。mixedmode表示虚拟机运行在混合模式下，即同时使用解释执行和即时编译（JIT）执行Java代码。Asharing表示虚拟机使用了类数据共享（CDS）功能，这可以提高启动速度和减少内存占用。
```
以上三行信息，我们一般只看第一行，知道自己安装的是哪个版本即可，版本只看前面的大版本
## 3— win10的bug
当电脑重启之后，环境变量失效了。表示操作系统不支持自定义的环境变量。
步骤:
* 还是要配置JAVA_HOME给以后的相关软件去使用
* 我们可以把JDK的完整路径配置到PATH当中。E:\develop\JDK\bin
